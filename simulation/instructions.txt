# containerlab :
To create the topography : sudo containerlab deploy --topo firewall.clab.yml
To properly destroy the topography : sudo containerlab destroy --topo firewall.clab.yml

containerlab graph --topo firewall.clab.yml
=> Render a graph from running lab or topo file if lab is not running, using HTML graph option with default server address :50080

To execute a command in a container : docker exec --user=root -it clab-firewallLab-node + commande (for instance ping, ...)

To access the bash of a client : sudo docker exec --user=root -it clab-firewallLab-client /bin/sh
To access the bash of the firewall : sudo docker exec --user=root -it clab-firewallLab-firewall /bin/bash

To push an image on a docker hub repo, there are 3 steps :
                                                - sudo docker login --username username
                                                - sudo docker tag my-image username/my-repo:my-image
                                                - sudo docker push username/my-repo:my-image

# Mosquitto library : 
sudo docker exec -it clab-firewallLab-client1 mosquitto_sub -h 10.10.4.2 -t '#'
=> subscribes to all topics on the broker with the wildcraft #

sudo docker exec -it clab-firewallLab-client2 mosquitto_pub -h 10.10.4.2 -t 'hello' -m 'world'
=> publishes 'world' with the topic 'hello' on the broker. Json data can also be published instead of strings, for instance "{\"value1\":20,\"value2\":40}"

For the authentication, publishes with arguments -u user and -P password or even --cafile certificat. 
=> for more details check http://www.steves-internet-guide.com/mosquitto_pub-sub-clients/

# Firewall :
To block all traffic with iptables : iptables/ip6tables -P INPUT/FORWARD/OUTPUT DROP
To enable all traffic with iptables : iptables/ip6tables -P INPUT/FORWARD/OUTPUT ACCEPT
